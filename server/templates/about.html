{{ _("Information about the technical stack and related choices") }}

{{ _("Back-end technologies") }}

{{ _("We chose python for our backend technology, along with the Flask framework. It’s a blazing fast and light plugin allowing us to handle basic routing and http requests. it also including the Jinja2 templating language.") }}

{{ _("League of legends is a world-class game. It was obvious that the i18n support was mandatory in order to reach a lot of players. For the sake of the demo, we only implemented the english and french language.") }}

{{ _("We rely on gettext through the Flask-Babel plugin and use Poedit to handle .po/.mo files.") }}

{{ _("Having a cache system is essential to avoid querying Riot’s server too much. Python dict/list are not persistent if the server crash so we chose not to use them.") }}

{{ _("Instead we chose to store response FROM the API in the key-value cache store Redis, a NoSQL DB. In Lol-Senpai, the key is the URL and the value is the HTTP response (body).") }}

{{ _("The caching strategy is very simplistic at the moment :") }}

  • {{ _("15 min by default") }}
  • {{ _("1 week for static data (champions’ info for instance) or summoner ids") }}
  • {{ _("few minutes for current games") }}

{{ _("The aim is to rely on the cache as much as possible.") }}

{{ _("We are also considering storing long term information on our server. Still using NoSQL technologies (maybe MongoDB), to provide more information and data for other modes and not only ranked mode.") }}

{{ _("Hosting technologies") }}

{{ _("In order to install, configure and launch these technologies on our server, we chose Docker. It allows us to automate every steps of the build..") }}

{{ _("Our flask application runs on nginx (web server). Unfortunately nginx (contrary to apache) does not support WSGI and thus cannot communicate with flask. That’s why Gunicorn comes into play, to act as a interface between our application and the web server.") }}

{{ _("Finally Supervisor monitor and control Gunicorn.") }}

{{ _("At the moment, we are hosting Lol Senpai on a cheap VPS. This is enough for now but this won’t hold the charge if ever get a small traffic. The next step is going to be the transfer on DigtalOcean who offers nice ssd servers for a cheap price.") }}

{{ _("Regarding our architecture, it’s hopefully, it’s going to be multi-servers. We are going to use :") }}

  • {{ _("HAProxy (acting as a load balanced)") }}
  • {{ _("Redis sentinel for redis replication") }}

{{ _("With this architecture, we believe that this would allow us to add new servers easily if we ever need to. And of course, we would need better servers and would migrate to AWS or OVH.") }}

{{ _("Like every developers these days, we use a CVS to work as a team. We chose git because it’s really powerful (despite not using branches for this contest haha).") }}
